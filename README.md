# 北海道 8 天 7 夜旅行網站

以手機操作為主的北海道自由行旅遊網站，行程日期為 2026/11/21–11/28，主要城市包含札幌、函館、登別與小樽。

- 線上網站：[caocharles.github.io/2026-sapporo-trip](https://caocharles.github.io/2026-sapporo-trip/)
- 主要入口：`index.html`
- 部署方式：GitHub Pages（`main` 分支）

## 網站特色

- Mobile-first 單頁式 App 介面
- 固定底部導覽列與 Hash 路由
- 每日時間軸、交通、住宿與備案整合
- 景點照片輪播、日文名稱、地址、語音及導航
- 即時天氣、互動地圖與步行路線
- 中／日／英旅遊會話與瀏覽器日文語音
- 行前清單、完成度、重要預訂與個人備忘
- 資料保存在使用者自己的瀏覽器，不需登入或後端服務

## 六個頁籤

| 頁籤 | 網址 | 內容與功能 |
| --- | --- | --- |
| 旅程總覽 | `#/overview` | 行程總覽圖、抵達與回程航班、八日速覽、六間住宿一覽。飯店卡可開啟中／日／英文名稱、日文地址、電話、入住退房時間、叫車資訊、撥號及導航。 |
| 每日行程 | `#/plan` | D1–D8 日期切換、概略時間、交通移動、行程提醒、住宿、景點、餐飲、柯南景點及備案。景點詳情提供照片、建議停留、日文、地址、Google Maps 與步行導航。 |
| 景點導覽 | `#/guide` | 依札幌、函館、登別、小樽篩選景點。重點景點提供多張照片、最近車站、冬季提醒、建議時段、景點故事與冷知識。季節推薦也可開啟詳情。 |
| 旅程地圖 | `#/map` | 依日期顯示最近車站至景點的步行路線、距離與時間。支援目前位置、日文播放、名稱複製、Google Maps 與步行導航。 |
| 旅行清單 | `#/lists` | 行前完成度、重要預訂、分類 Checklist、自訂項目及旅行備忘。完成狀態儲存在目前瀏覽器。 |
| 實用資訊 | `#/info` | 札幌、函館、登別與小樽即時天氣；依實際行程設計的中／日／英旅遊會話；旅遊注意事項與緊急聯絡資訊。 |

切換頁籤時，網址與瀏覽器標題會同步更新。重新整理會停留在原頁籤，也能使用瀏覽器上一頁／下一頁切換。

## 每日行程

| 日期 | 城市與主題 |
| --- | --- |
| D1・11/21 | 抵達札幌、中島公園、大通公園與聖誕市集 |
| D2・11/22 | 札幌前往函館湯之川、溫泉與熱帶植物園 |
| D3・11/23 | 函館兩日彈性第 1 日、五稜郭與元町地區 |
| D4・11/24 | 函館兩日彈性第 2 日、未完成景點與夜景備案 |
| D5・11/25 | 函館前往登別、熊牧場、地獄谷與溫泉街 |
| D6・11/26 | 登別前往小樽、小樽運河與夜間點燈 |
| D7・11/27 | 小樽返回札幌、北海道大學與市區景點 |
| D8・11/28 | 札幌前往新千歲機場，15:20 起飛返台 |

函館 D3–D4 為同一組彈性行程，會依天氣、能見度、體力及前一天完成狀況調整。

住宿詳情可直接複製「日文飯店名＋日文地址」給計程車司機，並提供電話撥打、Google Maps、步行導航與官方網站入口。飯店資料以各飯店官方網站為核對來源。

## 景點詳情與照片

行程卡與導覽卡使用共用的底部詳情頁，依資料完整度顯示：

- 中文與日文名稱
- 多張照片左右滑動
- 圖片頁數及授權來源
- 建議停留時間與類型
- 詳細日文地址及一鍵複製
- 日文語音播放與名稱複製
- Google Maps 地點與步行導航
- 冬季提醒、景點故事與冷知識

照片優先使用 Wikimedia Commons 等可確認授權的素材；無法確認授權或地點的圖片不會混用，頁面會顯示尚未找到可靠實景照片。

## 旅遊會話

旅遊會話依本次旅程規劃為七類，共 46 句：

- JR／函館市電
- 問路與景點
- 叫車
- 住宿
- 餐廳與候位
- 溫泉
- 求助與醫療

每句包含繁體中文、日文與英文，並標示適用場景。日文可直接播放或複製給對方查看。語音使用瀏覽器原生 Web Speech API，實際聲音依裝置可用的日文語音而異。

## 旅行清單

清單分為：

- 重要預訂：住宿、特急北斗、登別轉乘及回程交通
- 證件與付款
- 交通與網路
- 北海道冬季用品
- 隨身用品
- 出發當日確認
- 自訂項目
- 個人備忘錄與快速標籤

舊版 `checks` 清單會在第一次開啟新版時嘗試轉換到 `trip_list_v2`。資料只存在目前瀏覽器的 `localStorage`；更換裝置、清除網站資料或使用無痕模式時不會自動同步。

## 本機預覽

可直接開啟 `index.html`，或在專案目錄啟動靜態伺服器：

```bash
python3 -m http.server 8000
```

接著瀏覽：

```text
http://localhost:8000/
```

使用靜態伺服器預覽，能更接近 GitHub Pages 的網路請求與路由行為。

## 部署

網站由 GitHub Pages 直接發布 `main` 分支。更新流程：

1. 修改並檢查 `index.html` 與相關素材。
2. 同步更新本 README。
3. Commit 並 push 至 `origin/main`。
4. 等待 GitHub Pages 約 1–3 分鐘更新。
